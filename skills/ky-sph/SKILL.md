# ky-sph · 视频号 / 公众号原料

**核心：你已拿到的微信侧内容 → 本地可复用原料。**

本 skill **不内置、不捆绑任何第三方下载器**。  
获取原始视频或文章的方式由你自己选择（微信客户端另存、自有工具、人工复制文案等）。  
Agent 负责的是：**登记来源、落盘结构、转写/蒸馏下一步**。

## 触发后怎么做

1. 问清用户手里有什么（本地视频路径 / 口播文案 / 公众号正文 / 仅分享说明）。  
2. **不要**要求用户必须安装某个指定开源项目才能继续。  
3. 按下面「入库」写入用户指定目录（默认可提示 `信息搜集工作台` 或当前工作区）。  
4. 需要口播时：有本地音视频 → 指引本机转写；只有文案 → 直接蒸馏。  
5. 需要框架/话术时：套用户工作台的蒸馏模板（若有），否则输出结构化摘要。

## 入库约定（建议）

```text
{工作区}/01-源库/视频号/   或  公众号/
  YYYY-MM-DD_作者_短标题.md
  _raw/   # 可选：原始视频/截图
```

Markdown 最小 frontmatter：

```yaml
---
platform: weixin_channels   # 或 wechat_mp
author: ""
source_note: ""             # 用户描述的来源，勿编造链接
title: ""
date_captured: YYYY-MM-DD
media_path: ""              # 本地路径，可空
transcript_status: pending  # pending | done | n/a
---
```

正文：标题、摘要、口播/文章正文、备注。

## 获取素材时的诚实指引（通用）

若用户还**没有**文件、只有想法：

- 说明：微信生态多数内容需在 **本机客户端** 侧完成保存，Agent 无法代替操作微信窗口。  
- 给出可选方向（任选其一，**不点名捆绑具体第三方品牌/仓库作为本 skill 组件**）：  
  - 客户端内另存 / 转发到文件传输助手再导出  
  - 复制文案、字幕、口述要点  
  - 用户自己已在用的本机捕获工具  
- 用户补齐路径或文案后，再继续入库。

## 禁止

- 不把外部开源项目写成「本 skill 自带依赖」或「安装我们的下载器」。  
- 不协助绕过付费墙、破解、批量侵权搬运。  
- 不把他人项目 README/品牌贴进本仓库当自己的功能列表。
