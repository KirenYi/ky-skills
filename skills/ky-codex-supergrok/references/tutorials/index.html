30 秒总览

菜单怎么选（看清标注）：

  • ① OpenAI / ChatGPT 官方额度：ChatGPT 登录，不填 Key
  • ② SuperGrok（X 订阅）：grok login，不填 Key（Premium+ 别浪费 Grok 额度）
  • ③ DeepSeek … ← 官网 Key：去 platform.deepseek.com 申请，不要填 OpenRouter
  • ④ OpenRouter / Claude / Gemini：去 openrouter.ai 申请 sk-or-v1-…，不是 DeepSeek 官网 Key

记一句：DeepSeek 选带「官网 Key」的项；Claude/Gemini 选带「OpenRouter」的项。

配置 Claude / Gemini（OpenRouter Key）

  1. 浏览器打开 openrouter.ai/keys，登录后 Create Key，复制 sk-or-v1-...
  2. 双击「Codex 切换模型」，选带 OpenRouter 标注的 Claude/Gemini（或「④ OpenRouter 合集」）。弹窗里粘贴 sk-or-v1-…，不要填 DeepSeek 官网 Key。
  3. 等待 ChatGPT/Codex 自动重启，然后 新建对话（不要用旧会话）。
  4. 若已进入「API 模型」通道：可在 Codex 里用模型列表在 Claude / DeepSeek / Gemini 之间切换（桌面端有时显示「自定义」，属正常）。

手动写入 Key（可选）：

nano ~/.codex/ky-provider.env

# 文件内容仅一行即可：
OPENROUTER_API_KEY=sk-or-v1-你的密钥

# 保存后：
chmod 600 ~/.codex/ky-provider.env

配置 SuperGrok（订阅，不是 API 充值）

  1. 安装并打开 Grok Build CLI，执行：
    grok login
  2. 双击 「Codex → Grok」 或在「Codex 切换模型」里选 SuperGrok。
  3. 新建对话使用。若 502：终端执行 ~/.codex/bin/supergrok-proxy restart

日常怎么切换（越简单越好）

  • 最省事：双击桌面 Codex 切换模型 → 点一下想要的通道
  • 快捷按钮：Codex → OpenAI / Grok / Claude / DeepSeek / Gemini
  • 命令行：
    ~/.codex/bin/codex-provider pick
~/.codex/bin/codex-provider use deepseek
~/.codex/bin/codex-provider status

第一次配 Key 要粘贴一次；之后同一通道再点 = 直接切换。

配置 DeepSeek 官网 API（不是 OpenRouter）

  1. 打开 platform.deepseek.com/api_keys，登录并创建 API Key。
  2. 双击「Codex 切换模型」→ 选 DeepSeek Chat。弹窗里粘贴 Key →「保存并切换」。
  3. Key 保存在本机 ~/.codex/ky-provider.env 的 DEEPSEEK_API_KEY=...。
  4. 本机会启动本地协议桥（把 Codex 的 Responses 转成 DeepSeek Chat）。完全退出并重开 Codex，新建对话。

注意：DeepSeek 官网 Key 与 OpenRouter 的 sk-or-v1-... 不是同一种。Claude/Gemini 仍用 OpenRouter；DeepSeek 用官网 Key。

配置中转站 / 自定义 API（Claude 等）

  1. 在菜单选 ⑤ 中转站 / 自定义 API。
  2. 按提示填写：Base URL（如 https://你的中转.com/v1）、API Key、默认模型名（按中转站文档，如 claude-sonnet-4）。
  3. 本机保存到 ~/.codex/ky-provider.env，并启动本地协议桥（兼容 Codex Responses）。
  4. 完全退出并重开 Codex，新建对话测试。

说明：中转站只要提供 OpenAI 兼容的 Chat Completions（/v1/chat/completions）即可。Claude / GPT / DeepSeek 等由中转站侧决定，模型名填他们给你的那个字符串。

常见问题

Q：右下角能像官方模型那样点选吗？
A：在 API 模型（OpenRouter） 通道下，列表里会有 Claude/DeepSeek/Gemini 等；桌面端显示可能是「自定义」。SuperGrok 与官方 OpenAI 是不同通道，需用切换器切换。

Q：会不会改坏 Codex？
A：只改 ~/.codex/ 用户配置与本地脚本，不改 ChatGPT.app 本体。可随时切回 OpenAI。

Q：Key 存在哪？安全吗？
A：仅本机 ~/.codex/ky-provider.env（权限 600）。不要发到聊天或 GitHub。

Q：模型说自己是 GPT-5？
A：Codex 系统提示会带偏自称。以左下角通道 / codex-provider status 为准。
