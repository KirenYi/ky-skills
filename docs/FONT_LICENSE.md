# 字体许可约束（KY 硬性规则）

> **所有会进入对外内容的字体，必须公开可得、允许商用、且不把 KY / 用户置于字体侵权风险中。**  
> 适用于：`ky-cover`（未来）、`ky-mphtml`、封面测试导出、视频缩略图、文章头图等。

**作者**：Kiren Yi（KY）  
**状态**：生效中（2026-07-12）

---

## 1. 原则

1. **默认拒绝「看起来好看但不清楚授权」的字体。**  
2. **优先使用**：开放字体许可证（OFL / Apache 2.0 / SIL 等）或系统自带界面字体栈（不封装、不二次分发字体文件）。  
3. **禁止**：未授权的商用字库（常见盗版打包、来路不明的「免费商用」网盘包）、仅个人免费不可商用的字体。  
4. 若无法确认许可证 → **不用**，换已认证字体。

---

## 2. 推荐白名单（可默认使用）

### 2.1 中文（首选）

| 字体 | 许可（概要） | 用途建议 |
|------|----------------|----------|
| **Noto Sans SC**（思源黑体 / Source Han Sans 的 Noto 线） | SIL OFL / 相关开源许可 | 无衬线正文、标题、封面大字 |
| **Noto Serif SC**（思源宋体线） | SIL OFL 等 | 人文、长文、金句卡 |
| **Source Han Sans / Source Han Serif** | SIL OFL / Adobe 开源条款 | 同 Noto，专业排印 |
| **LXGW WenKai（霞鹜文楷）** | SIL OFL | 手写感、温和人文（注意辨认度） |

> 使用前以官方仓库/发布页的 **LICENSE 原文** 为准（名称随版本可能微调）。

### 2.2 西文 / UI

| 字体 | 许可（概要） | 用途建议 |
|------|----------------|----------|
| **Inter** | SIL OFL | 现代无衬线 |
| **IBM Plex Sans / Serif** | SIL OFL | 偏专业、科技 |
| **Source Code Pro** / **JetBrains Mono**（注意各版本许可） | OFL 等 | 仅代码展示 |
| 系统栈：`-apple-system`, `BlinkMacSystemFont`, `Segoe UI`, `Roboto` | 随 OS 预装 | **仅作 CSS `font-family` 回退**，不打包字体文件 |

### 2.3 CSS 推荐写法（公众号 HTML / 网页）

```css
/* 不 @font-face 外链时：设备字体 + 安全回退（不二次分发字体文件） */
font-family: "Noto Sans SC", "Source Han Sans SC",
  -apple-system, BlinkMacSystemFont, "Segoe UI",
  "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
  sans-serif;
```

说明：

- 在用户手机/电脑上调用**本机已装字体**做显示，一般不构成「分发字体文件」。  
- 若我们的工具 **下载并内嵌** `.ttf/.otf` 到安装包或导出 ZIP，则必须只内嵌白名单中的 OFL/Apache 字体，并附带许可证副本。

---

## 3. 禁止 / 高风险（默认不用）

- 未购买授权的方正、汉仪、造字工房等**商用字库**（除非 KY 已购买并确认授权范围覆盖用途）。  
- 网盘「全套字体包」「破解字体」。  
- 仅「个人免费、商用需购买」且我们未购买的字体。  
- 来源不明的「抖音同款字体」文件。

---

## 4. 按产出物怎么做

| 产出 | 字体策略 |
|------|----------|
| **公众号 HTML**（粘贴后台） | 用系统/通用中文字体栈；不依赖必须外链 Google Fonts（公众号常拦外链） |
| **封面 PNG/JPG**（代码渲染） | 渲染机可装 Noto / Source Han；或仅用系统栈光栅化——**导出的是位图，不附带字体文件** |
| **可下载模板包**（若含字体） | 只含 OFL/Apache 字体 + `LICENSE` 文本 |
| **AI 生图里的字** | AI 易乱码；KY 封面**优先代码排字**。若 AI 图带字，字仍应短，且最终以可商用流程复核 |

---

## 5. Agent / 开发者检查清单

做封面或改 `ky-mphtml` 样式前：

- [ ] 用到的每一个 `font-family` 名称，是否在白名单或系统回退栈内？  
- [ ] 是否引入了 `@font-face` 指向本地/网盘不明字体？→ 禁止  
- [ ] 若新增字体，是否已记录：名称、下载官方源、许可链接、用途？  
- [ ] 交付物是否误带了 `.ttf` 进 git？→ 不应提交未授权字体文件  

新增字体时，在本文件「白名单」表追加一行，并更新 `docs/PROGRESS.md`。

---

## 6. 与封面测试的关系

`ky-封面测试` 当前 HTML 使用系统中文字体栈（PingFang / Microsoft YaHei 等）做**本机预览与 PNG 导出**，不打包字体文件，符合「不二次分发」策略。

后续正式 `ky-cover` skill 将：

1. 默认排字字体：**Noto Sans SC / Noto Serif SC**（可商用 OFL 线）  
2. 继续禁止未授权商用字库  
3. 人像出镜版：照片素材需用户自有版权，与字体规则分开约束
